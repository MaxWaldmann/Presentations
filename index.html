Маючи зважений граф (орієнтований або неорієнтований) та вихідну вершину-джерело, необхідно знайти найкоротші шляхи від неї до всіх інших вершин графа. Це задача SSSP (Single Source Shortest Path).

navigation

Аналогія з картами (Google Maps, Uber)

Навігатор уявляє місто як складну мережу: перехрестя є вершинами, а дороги між ними — ребрами. Вага кожного ребра визначається відстанню, станом покриття та поточними заторами.

Едсгер Дейкстра

(1930 – 2002) Нідерланди

Філософія "Егоїста"

Це жадібний алгоритм (Greedy). На кожному кроці він обирає найближчу ще не відвідану вершину, фіксує шлях до неї як остаточний та проводить релаксацію її сусідів. Він «спалює мости» заради миттєвої вигоди.

⚠️ Головне обмеження

Алгоритм Дейкстри коректно працює тільки для графів без ребер з від'ємними вагами: w(u, v) ≥ 0.

Якщо в графі є від'ємні ваги, жадібний вибір фіксує оцінку шляху передчасно, позбавляючи можливості знайти дешевший обхідний шлях пізніше.

info Історія створення (20 хвилин) expand_more
Дейкстра вигадав цей алгоритм у 1956 році під час кавування на терасі кафе зі своєю нареченою. Йому потрібно було продемонструвати можливості нового комп'ютера ARMAC. Він обрав задачу пошуку найкоротшого шляху на дорожній карті Нідерландів від Роттердама до Гронінгена і розв'язав її у своїй голові за 20 хвилин без використання паперу та олівця.

code Псевдокод алгоритму Дейкстри

dijkstra(G, start) 
   function Dijkstra(Graph, source):
    
       for each vertex v in Graph.Vertices:
           dist[v] ← INFINITY
           prev[v] ← UNDEFINED
           add v to Q
      dist[source] ← 0
     
       while Q is not empty:
          u ← vertex in Q with minimum dist[u]
          Q.remove(u)
         
          for each edge (u, v) in Graph:
              alt ← dist[u] + Graph.Distance(u,v)
              if alt < dist[v]:
                  dist[v] ← alt
                  prev[v] ← u

      return dist[], prev[]

Складність алгоритму

З масивом: O(V2)
З двійковою купою: O(E log V)
З купою Фібоначчі: O(E + V log V)

terminal Практична реалізація Дейкстри

dijkstra_opt.cpp 
#include <iostream>
#include <vector>
 
using namespace std;
 
const int INF = 1e9; // Нескінченність
 
void Dijkstra(int vertices, int source, const vector<vector<pair<int, int>>>& graph) {
    // Ініціалізація (рядки 3-7 псевдокоду)
    vector<int> dist(vertices, INF);
    vector<bool> visited(vertices, false); // Виступає в ролі перевірки наявності в черзі Q
    
    dist[source] = 0;
 
    // Головний цикл (рядки 9-17)
    for (int i = 0; i < vertices; i++) {
        // Знаходимо невідвідану вершину з найменшою відстанню (рядок 10)
        int u = -1;
        for (int j = 0; j < vertices; j++) {
            if (!visited[j] && (u == -1 || dist[j] < dist[u])) {
                u = j;
            }
        }
 
        // Якщо найближча вершина недосяжна, перериваємо (запобіжник)
        if (dist[u] == INF) break; 
 
        // Видаляємо вершину з черги невідвіданих (рядок 11)
        visited[u] = true;
 
        // Перевіряємо всіх сусідів обраної вершини (рядок 13)
        for (const auto& edge : graph[u]) {
            int v = edge.first;     // сусід
            int weight = edge.second; // вага дороги до сусіда
 
            // Якщо знайшли коротший шлях — оновлюємо (рядки 14-17)
            if (dist[u] + weight < dist[v]) {
                dist[v] = dist[u] + weight;
            }
        }
    }
 
    // Відладка: вивід результату
    cout << "Дейкстра: Найкоротші шляхи від вершини " << source << ":\n";
    for (int i = 0; i < vertices; i++) {
        cout << "До вершини " << i << " відстань: " << dist[i] << "\n";
    }
}
 
int main() {
    int V = 5; // Кількість вершин
    
    // Створюємо граф у вигляді списку суміжності: гра[звідки] = {куди, вага}
    vector<vector<pair<int, int>>> graph(V);
    graph[0].push_back({1, 10});
    graph[0].push_back({4, 5});
    graph[1].push_back({2, 1});
    graph[1].push_back({4, 2});
    graph[2].push_back({3, 4});
    graph[3].push_back({2, 6});
    graph[3].push_back({0, 7});
    graph[4].push_back({1, 3});
    graph[4].push_back({2, 9});
    graph[4].push_back({3, 2});
 
    Dijkstra(V, 0, graph);
    return 0;
}

Алгоритм Дейкстри ("Егоїст" у пошуках найшвидшого шляху)

Уяви, що ти стоїш у своєму місті (Старт) з блокнотом. У блокноті записані всі міста України, і навпроти кожного стоїть час у дорозі.

Базується на ідеї динамічного програмування. Він діє обережно та методично. Замість вибору найкоротшого ребра він робить V - 1 повних проходів по всіх ребрах графа, оновлюючи відстані. Він нікому не довіряє наперед.

✔️ Переваги підходу

На відміну від Дейкстри, Беллман-Форд легко працює з негативними вагами та вміє детектувати небезпечні від'ємні цикли.

info Звідки взявся цей метод? expand_more
Річард Беллман сформулював концепцію динамічного програмування у RAND Corporation. Йому довелося вигадати таку незвичну назву, щоб приховати від міністра оборони США, який не любив математику, теоретичний характер досліджень. Слово «програмування» тоді означало просто планування, а «динамічний» звучало прогресивно. Лестер Форд доповнив систему ідеєю покрокової релаксації.

Складність алгоритму

Часова складність: O(V · E)
Просторова складність: O(V)

Принцип V-1 кроків:

Будь-який найкоротший простій шлях мітримує не більше V - 1 ребер. Якщо після V - 1 ітерацій ми знову можемо релаксувати хоч одне ребро — граф містить від'ємний цикл. Шлях через такий цикл дорівнюватиме мінус нескінченності.

terminal Практична реалізація Беллмана-Форда

bellman_ford.cpp 
#include <iostream>
#include <vector>

using namespace std;

const int INF = 1e9; // Встановлюємо значення нескінченності

// Структура для зберігання ребра (звідки, куди, вага)
struct Edge {
    int u, v, weight;
};

void BellmanFord(int vertices, int source, const vector<Edge>& edges) {
    // 1. Ініціалізація: всі відстані = ∞, крім початкової
    vector<int> dist(vertices, INF);
    dist[source] = 0;

    // 2. Основна частина: розслаблення всіх ребер (V - 1) разів
    for (int i = 1; i <= vertices - 1; i++) {
        for (const auto& edge : edges) {
            // Якщо шлях до u відомий і через u до v ближче, ніж напряму:
            if (dist[edge.u] != INF && dist[edge.u] + edge.weight < dist[edge.v]) {
                dist[edge.v] = dist[edge.u] + edge.weight; // Оновлюємо відстань
            }
        }
    }

    // 3. Перевірка на наявність циклів з від'ємною вагою
    for (const auto& edge : edges) {
        if (dist[edge.u] != INF && dist[edge.u] + edge.weight < dist[edge.v]) {
            cout << "Помилка: У графі є цикл з від'ємною вагою!" << endl;
            return; // ХИБА
        }
    }

    // Відладка: вивід результату (ІСТИНА)
    cout << "Беллман-Форд: Найкоротші шляхи від вершини " << source << ":\n";
    for (int i = 0; i < vertices; i++) {
        cout << "До вершини " << i << " відстань: " << dist[i] << "\n";
    }
}

int main() {
    int V = 5; // Кількість вершин
    vector<Edge> edges = {
        {0, 1, 6}, {0, 3, 7}, {1, 2, 5}, 
        {1, 3, 8}, {1, 4, -4}, {2, 1, -2}, 
        {3, 2, -3}, {3, 4, 9}, {4, 0, 2}, {4, 2, 7}
    };

    BellmanFord(V, 0, edges);
    return 0;
}

Алгоритм Беллмана-Форда ("Параноїк" із нескінченними перевірками)

Тут підхід зовсім інший. У нас є список абсолютно всіх існуючих доріг у країні.

1. Підготовка

bellman_ford_init.cpp 
vector<int> dist(vertices, INF);
dist[source] = 0;
  • Переклад для мозку: Те саме. Блокнот, всюди нескінченність, вдома — нуль.

2. Тотальна перевірка (Головна частина)

bellman_ford_relax.cpp 
for (int i = 1; i <= vertices - 1; i++) {
    for (const auto& edge : edges) {
        if (dist[edge.u] != INF && dist[edge.u] + edge.weight < dist[edge.v]) {
            dist[edge.v] = dist[edge.u] + edge.weight;
        }
    }
}
  • Переклад для мозку: Беллман-Форд не ходить по містах, він сидить в офісі. Він бере список УСІХ доріг і перевіряє кожну: "А чи не можу я через цю дорогу прокласти коротший шлях?". Він робить це для кожної дороги. Але є нюанс! Оскільки він сидить в офісі, інформація про короткі шляхи поширюється повільно. Тому він бере і перечитує весь цей гігантський список доріг знову. І знову. І так стільки разів, скільки всього міст на карті. Це довго, але він гарантує, що не пропустить жодної вигоди.

3. Перевірка на "збій у матриці" (Цикли від'ємної ваги)

bellman_ford_check.cpp 
for (const auto& edge : edges) {
    if (dist[edge.u] != INF && dist[edge.u] + edge.weight < dist[edge.v]) {
        cout << "Помилка: У графі є цикл з від'ємною вагою!" << endl;
        return;
    }
}
  • Переклад для мозку: Після того як він перечитав список купу разів, він робить це ще один, контрольний раз. Якщо після всіх ідеальних прорахунків він знову знаходить коротший шлях, він розуміє: щось не так! Значить, десь на карті є "магічне кільце" (від'ємні ваги), катаючись по якому, ти не витрачаєш час, а здобуваєш його нескінченно. В реальності такого не буває, тому алгоритм кричить "Помилка!".

Ось так це працює. На захисті ти можеш просто вказувати пальцем (або мишкою) на відповідні блоки коду і говорити ці прості "переклади". Викладач оцінить саме те, що ти розумієш суть, а не зубриш символи.

Як тобі таке пояснення, чи стало трохи легше уявити, як ці алгоритми "бігають" по карті?

На звичайній дорожній карті немає доріг із від'ємним часом чи відстанню (неможливо приїхати в місто раніше, ніж ви виїхали). Тому логістичні сервіси використовують модифікації швидкодіючого Дейкстри (алгоритм A* та Road Hierarchies).

У комп'ютерних мережах протокол OSPF використовує Дейкстру для локальної карти кожного роутера. Протокол RIP використовує Беллмана-Форда для обміну векторами відстаней між сусідами без знання повної топології.
